World Monitor：把全球态势感知塞进浏览器的开源 OSINT 仪表盘

发表于 2026-07-22 分类于 AI 开源项目
koala73 开源的 World Monitor：500+ 信息流、56 地图层、CII v8 国家不稳定性评分、39-tool MCP server，单 docker compose 启动，把 Bloomberg 级情报视图搬到任何浏览器。

开头：信息过载的真实痛感

你有没有过这种体验 —— 周五晚上你打开 Bloomberg、Reuters、新华、Telegram 群、Twitter、Substack、各种 RSS，几乎每条都在喊” 大事来了”，但你脑子里只有一句话：到底哪些是真的重要的？哪些之间是相互关联的？哪些会在下周一影响我的代码、我的仓位、我的工作？

真相是：现在全球每天产生的公开新闻、卫星过境、ADS-B 航班、电网异常、油轮轨迹、社交媒体信号加起来超过 10TB，但绝大多数人打开浏览器看到的还是一长串时间线加一个静态地图。每条信号都在喊” 看我”，但没人告诉你哪些信号在同一个故事里。

koala73/worldmonitor（GitHub Trending #1，1,295 stars 当日涨幅、累计 62.5k stars）就是为了把这堆” 碎信号” 压成一张可操作的态势图。今天这篇文章带你拆解一下这个开源项目到底做了什么、有多硬核、你能怎么用。

项目背景：把” 全球态势感知” 从政府机柜搬到浏览器

World Monitor 是 Elie Habib 自 2024 年开始的单人 / 百人协作项目，主仓库在 github.com/koala73/worldmonitor。它的目标不是再做一款” 新闻聚合器” 或者” 地图工具”—— 它要做的是专业级 OSINT（Open Source Intelligence，开源情报）仪表盘，把政府 / 对冲基金 / 智库日常用的那种” 实时态势感知” 能力，免费搬到任何有浏览器的电脑上。

为什么是 2026 年才有可能？因为三件事同时到位了：

  • WebGL 性能溢出 ——deck.gl + MapLibre GL 能在浏览器流畅渲染几万到几十万个矢量要素
  • 廉价 AI 推理 —— 本地 Ollama / LM Studio 让” 每天聚合 500 条新闻自动出摘要” 接近零成本
  • 开放数据爆炸 ——ADS-B、GPS jamming、SGP4 卫星过境、MarineTraffic 油轮、Solar 地磁、USGS 地震、FIRMS 野火…… 每一条数据流都开放，都可拉

World Monitor 把这三股力量压成” 一个仪表盘、六个变体”。它的产品形态是六个域名 + 一个 Tauri 桌面端：

变体 URL 关注
World worldmonitor.app 综合地缘 + 基础设施 + 财经
Tech tech.worldmonitor.app 技术公司、AI 模型、开源项目
Finance finance.worldmonitor.app 29 家交易所 + 大宗商品 + 加密 + 7-signal 合成
Commodity commodity.worldmonitor.app 大宗商品专项
Energy energy.worldmonitor.app 能源 + 电网 + LNG
Happy happy.worldmonitor.app 只推好消息（让人喘口气）

核心功能亮点

🌍 56 层地图 + 双引擎

打开 worldmonitor.app，第一个让你愣住的是那颗 3D 地球 ——globe.gl + Three.js 实时渲染，叠加 deck.gl + MapLibre GL 的 WebGL 平面图，可以在两个引擎之间秒切。

地图上同时叠加 56 种图层 —— 冲突区、军事基地、海底电缆、ADS-B 航班、SGP4 实时卫星过境（含 ISS / Starlink / 军用卫星）、GPS jamming 与 spoofing 区、MarineTraffic 油轮轨迹、USGS 地震、FIRMS 野火热点、NOAA 飓风、Solar 地磁风暴（Kp 指数）等等。所有图层都可以独立开关、调整透明度、设置地理过滤。

🧠 Country Instability Index（CII v8）

这是 World Monitor 最硬核的私有信号。CII 是一个服务端权威的、专有的国家不稳定性评分，覆盖 31 个 Tier-1 国家。每天根据公开数据自动打分，输出 0–100 的压力指数：

  • 政治维度 —— 议会动荡、罢工、抗议、突发戒严
  • 经济维度 —— 汇率跳水、CDS 利差、央行异常操作
  • 安全维度 —— 恐袭、军事调动、关键基础设施异常
  • 社会维度 —— 断网、通讯中断、大规模迁徙

CII v8 不是简单的关键词计数，它跑在一个 server-authoritative 的 pipeline 里，把多源信号做归一化、加权、衰减。dashboard 上每个国家的卡片用颜色编码压力等级，旁边直接列出” 是哪些信号把分数推上来的”—— 可追溯、可证伪。

📰 500+ 策展信息流 + AI 简报

World Monitor 内置 500+ 个策展信息源，覆盖 15 个分类（地缘、防务、能源、科技、金融、加密、气候……）。每天这些信息流跑过一遍 AI 摘要引擎（Ollama / Groq / OpenRouter 任选），输出” 今天的全球要点 + 5 条关联信号”。

更关键的是 cross-stream correlation（跨流关联）—— 比如一则” 红海航运中断” 的新闻、一组”Brent 油价跳涨” 的行情、一条” 苏伊士运河 GPS jamming 区域扩大” 的信号，会被自动归到同一个 story 卡片里，告诉你” 这三个东西是同一件事”。

📊 7-signal Market Composite

Finance 变体里有个叫 7-signal Market Composite 的东西 —— 把 7 个相互弱相关的市场信号（VIX、DXY、Brent、铜油比、信用利差、比特币、新兴市场货币）合并成一个 0–100 的合成指数，让你一眼看出” 市场现在是不是处在系统性紧张状态”，而不是盯死一个 VIX。

🛰️ ADS-B + SGP4 卫星过境

浏览器的 Transformers.js 侧跑 SGP4 轨道计算器 —— 点开卫星图层，你能实时看到 ISS、Starlink 链、Hubble、各国军用侦察卫星过境头顶的轨迹。ADS-B 模块则把全球商用航班位置（带高度、速度、航向）实时画出来。两套叠在一起，就是个迷你版的” 太空 + 空中态势图”。

🛠️ 39-tool MCP Server

World Monitor 给 AI agent 用。它暴露一个 Streamable HTTP MCP server（https://worldmonitor.app/mcp），包含 39 个工具 ——get_country_cii、list_hotspots、get_oil_supply_disruptions、get_satellite_passes…… 任何 MCP 兼容的 agent（Claude Code、Cursor、Codex）都能直接调用。

比如你可以让 Claude Code：

“用 worldmonitor MCP 帮我查一下过去 24 小时哪个 Tier-1 国家 CII 涨幅最大，给我三条关联新闻。”

它会真的拉数据、聚关联、输出结论。

实战示例：3 分钟跑起来

1. Web 版（最简单）

直接打开 https://worldmonitor.app，零注册、零账号、零信用卡。打开就是完整仪表盘。

2. 桌面端（Tauri 2）

桌面端和 Web 版共享同一份后端数据，但本地有缓存 —— 断网时仍能查看最近一次同步的状态。

3. CLI /npm 包

4. MCP 接入 Claude Code

然后在 Claude Code 里：

“用 worldmonitor 给我列一下 CII> 70 的国家，并标注最近 24h 涨幅最大的前 3 个。”

5. 本地 AI（Ollama）

之后所有 AI 简报都在本地跑，零 token 成本、零数据外泄。

6. Self-host（Docker）

仓库提供了完整的 docker-compose.yml、Dockerfile、Dockerfile.relay、Dockerfile.digest-notifications，支持自托管 + 反向代理 + 邮件摘要。

一句话总结：Bloomberg 卖你” 分析好的信息流”，World Monitor 给你” 造这个信息流的全部原料 + 引擎”。对个人开发者、独立研究员、技术决策者来说 —— 后者价值远大于前者，因为你能用 MCP 把它接进自己的 agent 工作流。

适用场景和限制

最适合：

  • 你做地缘 / 宏观 / 能源 / 科技政策研究，需要每天看” 全球在发生什么”
  • 你做 OSINT / 调查报道，需要把多源信号拼成故事
  • 你想给 Claude Code / Codex 接一个真实世界数据源 ——MCP server 39 工具直接用
  • 你做 side project，需要一个免费、零运营成本的” 全球仪表盘” 作为背景数据
  • 你做产品 / 投资决策，需要一个比 Twitter 时间线更严肃的全局视图

不太适合：

  • 想要” 个股深度财务分析”——World Monitor 不替代 Bloomberg 的公司层数据
  • 需要高频交易信号（毫秒级延迟）—— 它的刷新周期是分钟级，不是 tick 级
  • 需要企业内部网 / 专网数据 —— 它是公开数据聚合器，不是企业情报平台
  • 完全离线场景 —— 大部分数据流要联网，但你可以在 self-host 模式下加 24h 缓存

需要留意的坑：

  • CII v8 是 server-authoritative 的专有算法，自托管版本可能拿到的是上一日缓存，实时评分需要用云版
  • 56 个图层全开会吃显卡 —— 普通笔记本建议保持 ≤ 20 个图层
  • AI 简报默认走云端（Groq / OpenRouter），要零成本必须切到本地 Ollama
  • 自托管需要一定的反向代理 / Docker 经验，文档已经写得很细但仍要花半天

总结

World Monitor 不是又一款” 漂亮的 dashboard”—— 它是把” 全球态势感知” 这件事第一次以开源 + AGPL + 自托管的形式交到普通开发者手里。它把 500 条信息流、56 个地图层、一个 CII 评分算法、一个 39-tool MCP server 压进同一个仓库，然后让 Ollama / Groq / OpenRouter 任你选。

如果你最近被” 每天刷太多新闻源但拼不出一条完整故事” 困扰过，去 https://worldmonitor.app 打开它 ——3 分钟之内你就会明白这玩意儿和市面上所有” 信息聚合” 产品的差距在哪里。
