Marketing Skills：给 Claude Code 装上 47 项营销工作流，开发者也能系统做增长

发表于 2026-07-14 分类于 AI Agent ， 开源项目
一套兼容多种编码 Agent 的开源营销技能库，用共享产品上下文串起 SEO、文案、转化和增长工作流。

会写产品，却不知道怎么把它卖出去？

产品已经能跑，落地页也上线了，接下来该做什么？改标题、查 SEO、设计注册流程、写邮件，还是先研究客户？很多开发者并不缺执行力，真正缺的是一套能决定先做什么、做到什么程度、如何检查结果的方法。

直接问大模型 “帮我做营销”，常见结果是一份四平八稳的建议清单。它记不住产品定位，也不会主动把文案、转化率优化和数据埋点连成同一条链路。每次开新会话，还要重新解释用户是谁、产品解决什么问题。

今天 GitHub Trending 上的 coreyhaines31/marketingskills 正在解决这类问题。它不是另一个营销聊天机器人，而是一套遵循 Agent Skills 规范的开源工作流。写作时项目约有 38.6k stars，当天新增约 299 stars，最新提交就在数小时前，维护状态很活跃。

项目背景：把营销方法装进 Agent

Agent Skill 本质上是一个带 SKILL.md 的目录。文件用 YAML 描述名称和触发条件，用 Markdown 写清步骤、检查项、边界条件；需要时还能附带脚本、参考资料和模板。Agent 判断任务匹配后再加载它，不必把所有营销知识一次塞进系统提示词。

Marketing Skills 把这套机制用到了营销全流程。仓库当前包含 47 个技能目录，覆盖 SEO 审计、AI 搜索优化、文案、CRO、注册与 onboarding、广告、分析、定价、流失预防、销售支持和市场计划。它兼容 Claude Code、Codex、Cursor、Windsurf，以及支持 Agent Skills 规范的其他工具，许可证为 MIT。

它解决的核心问题不是 “让 AI 写得更多”，而是让 AI 按稳定流程工作：先读产品事实，再收集缺失信息，接着执行专项任务，收尾时按清单自检。

四个最有价值的设计

1. 用共享产品上下文减少重复解释

product-marketing 是整套技能的底座。它会建立 .agents/product-marketing.md，记录产品类型、目标客户、JTBD、痛点、竞争格局、品牌语气、证据和业务目标。其他技能开工前会先读这个文件，只询问当前任务缺少的信息。

这样做的好处很直接：同一个价值主张会贯穿落地页、邮件和广告，不容易出现首页面向开发团队，广告却突然对企业采购说话的割裂感。上下文是普通 Markdown，也方便团队审查和用 Git 管理。

2. 47 个技能覆盖完整漏斗

这不是只有几个文案模板的提示词包。seo-audit 会按技术、页面和内容问题检查站点；copywriting 强调客户语言、单一主行动和可验证证据；signup、onboarding、churn-prevention 分别处理注册、激活和留存；analytics 与 ab-testing 负责测量和实验。

技能之间还能互相引用。例如写落地页时，copywriting 会连接 cro 与 ab-testing；客户研究结果会进入文案、转化优化和竞品页面。营销动作不再是 47 个孤立按钮，而是可以组合的工作流。

3. 不只给答案，还规定检查方式

单次提示词常把 “输出一段文案” 当成终点。这里的技能会规定输入、步骤和验收。例如文案技能要求确认页面唯一主行动，优先使用客户原话，区分功能与收益，并检查空洞术语、被动语态和无法证明的数字。

这种约束不保证结果自动正确，却能降低漏项。团队也能直接修改 SKILL.md，把自己的审批规则、禁用词和行业要求加进去。

4. 安装轻，迁移成本低

技能主要是 Markdown 和引用资料，不依赖某个闭源营销平台。可以一次安装全部，也可以只选两三个高频能力。它不会代替现有 CMS、分析平台或广告账户，而是给操作这些工具的 Agent 提供方法层。

适用场景与限制

它很适合小型产品团队、技术型创始人、内容与增长负责人，也适合为多个项目建立统一营销流程。常见用途包括新产品定位、落地页改版、SEO 体检、邮件序列、发布计划和定期增长复盘。

它与同类方案的差异主要在三点：比零散提示词包更系统，有共享上下文和技能依赖；比单一 “AI CMO” 产品更透明，所有规则都能阅读和修改；比连接广告账户的 MCP 更偏方法论，不会自动替你操作外部平台。

限制也来自这里。它提供的是程序化知识，不自带 GA4、Search Console 或广告平台数据；模型质量和输入材料会直接影响结论；47 个技能全装并不等于应该同时启用。更稳妥的方式是从 product-marketing 加一两个具体技能开始，跑通审阅流程后再扩展。涉及预算、批量外发、隐私数据和公开承诺的动作，必须保留人工审批。

总结

Marketing Skills 最有价值的地方，不是把营销包装成一句神奇提示词，而是把专业人员反复使用的方法拆成可版本化、可组合、可检查的 Agent 工作流。它让编码 Agent 从 “偶尔帮忙写文案” 前进一步，开始围绕同一份产品事实协作。

如果你正维护一个已有代码仓库，可以先安装 product-marketing、copywriting 和 cro，用一次真实首页改版检验它。能减少多少重复沟通、能否让每条主张都找到证据，比安装了多少技能更重要。
